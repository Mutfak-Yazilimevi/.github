# Sample Eval Cases

## 1. Very Vague Request

- Input: "Make my page more premium."
- Expected detected mode: Mode B if an existing page is implied; ask for page context if none exists.
- Expected output focus: Clarify goal, diagnose likely hierarchy/structure issues, avoid decoration-first advice.
- Common mistakes: Adding gradients, glass, and animation without diagnosis.
- Recommended scoring focus: Intent detection, vague advice avoidance, open questions.

## 2. New SaaS Dashboard From Scratch

- Input: "I am building a SaaS analytics dashboard. Help me design the UI before coding."
- Expected detected mode: Mode A.
- Expected output focus: App shell, metrics, charts, tables, filters, states, responsive layout.
- Common mistakes: Creating a marketing landing page instead of an operational dashboard.
- Recommended scoring focus: Project type, IA, component system, frontend guidance.

## 3. Existing Landing Page Redesign

- Input: "This landing page feels generic and does not convert."
- Expected detected mode: Mode B.
- Expected output focus: Hero clarity, product proof, CTA hierarchy, trust, conversion flow.
- Common mistakes: Only changing colors or adding animations.
- Recommended scoring focus: Diagnosis, hierarchy fixes, before/after direction.

## 4. AI Tool Interface

- Input: "Design an AI writing tool with prompt input, model settings, history, and result preview."
- Expected detected mode: Mode A.
- Expected output focus: Prompt/result workflow, states, model controls, history, export/refine actions.
- Common mistakes: Making it a chatbot only or overusing dark neon visuals.
- Recommended scoring focus: Component system, motion, product realism.

## 5. Admin Dashboard Upgrade

- Input: "My admin panel is just tables and looks boring. Improve it but keep it usable."
- Expected detected mode: Mode B.
- Expected output focus: Data hierarchy, filters, summary cards, table states, detail drawer.
- Common mistakes: Reducing density too much or adding decorative charts.
- Recommended scoring focus: Product realism, implementation feasibility.

## 6. Personal Portfolio

- Input: "I want a premium designer/developer portfolio."
- Expected detected mode: Mode A.
- Expected output focus: identity, selected work, case studies, visual proof, contact.
- Common mistakes: Decorative animation without project proof.
- Recommended scoring focus: IA, visual system, anti-template guidance.

## 7. Mobile App Screen

- Input: "Plan a premium mobile home screen for my habit app."
- Expected detected mode: Mode A.
- Expected output focus: one primary action, status, touch targets, bottom nav, mobile states.
- Common mistakes: Desktop density, tiny controls, too many cards.
- Recommended scoring focus: Responsive/mobile behavior, interaction states.

## 8. Creator Tool Interface

- Input: "I want a modern creator tool for generating social media covers."
- Expected detected mode: Mode A.
- Expected output focus: workspace layout, canvas preview, controls, presets, history, export.
- Common mistakes: Treating it like a landing page.
- Recommended scoring focus: Project type, IA, component system.

## 9. Data Dashboard

- Input: "Make this data dashboard more professional and easier to scan."
- Expected detected mode: Mode B.
- Expected output focus: metric hierarchy, chart purpose, legends, filters, table density.
- Common mistakes: Adding chart decorations without improving decision flow.
- Recommended scoring focus: Information hierarchy, visual system, feasibility.

## 10. Pricing Page

- Input: "Design a pricing page that feels premium and trustworthy."
- Expected detected mode: Mode A.
- Expected output focus: plan comparison, recommended plan, feature clarity, FAQ, trust.
- Common mistakes: Over-highlighting every plan or hiding terms.
- Recommended scoring focus: CTA hierarchy, component states, clarity.

## 11. Product Settings Page

- Input: "Our settings page looks like a plain form. Make it product-ready."
- Expected detected mode: Mode B.
- Expected output focus: grouped settings, sidebar tabs, form states, save feedback, danger zone.
- Common mistakes: Adding visual effects instead of grouping and states.
- Recommended scoring focus: IA, component system, state coverage.

## 12. Dark Tech-style Homepage

- Input: "Create a dark tech-style homepage for an AI infrastructure product."
- Expected detected mode: Mode A.
- Expected output focus: product category, developer trust, technical proof, controlled dark style.
- Common mistakes: Neon overload, illegible text, generic AI orbs.
- Recommended scoring focus: Visual system, 3D restraint, implementation prompt.
