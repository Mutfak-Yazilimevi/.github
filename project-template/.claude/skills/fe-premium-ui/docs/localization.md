# Localization

This repository maintains multilingual README files by default. The GitHub default `README.md` is Simplified Chinese because the primary creator audience is Chinese AI Coding / Vibe Coding users.

## README Files

- `README.md`: Simplified Chinese default README shown by GitHub.
- `README.en-US.md`: English README for global open-source users.
- `README.zh-CN.md`: Simplified Chinese mirror README for direct language linking.
- `README.ja-JP.md`: Japanese README.
- `README.ko-KR.md`: Korean README.

## Consistency Rules

Core information must remain consistent across all README files:

- Same Skill name: `premium-ui-builder-skill`
- Same positioning
- Same two-mode logic
- Same file structure
- Same usage method
- Same license
- Same language navigation pattern
- Same badge and Star History sections

## Update Rules

When feature descriptions change:

1. Update `README.md`.
2. Update `README.en-US.md`.
3. Update `README.zh-CN.md`.
4. Update `README.ja-JP.md`.
5. Update `README.ko-KR.md`.
6. Confirm the mode descriptions still match `SKILL.md`.

## Tone Guidelines

- English: clear, open-source friendly, practical.
- Simplified Chinese: natural for AI creators and Vibe Coding users; beginner-friendly but not pure marketing copy.
- Japanese: concise, clear, and technically understandable.
- Korean: concise, clear, and technically understandable.

## Translation Quality

Translations do not need to be literal. They should preserve meaning, usage, and expectations. If a concept has no natural local equivalent, keep the English UI term and explain it simply.
