# Examples

These examples show how `premium-ui-builder-skill` turns vague UI goals into structured design and implementation briefs.

Each example includes:

- User original input
- Detected mode
- Project type
- UI goal
- Key design problems or assumptions
- Recommended UI strategy
- Information architecture
- Visual system
- Component system
- Motion strategy
- Frontend implementation recommendation
- Codex-ready prompt
- Acceptance criteria

Use them as reference patterns, not fixed outputs. A good response should adapt to the user's actual product, audience, stack, and existing UI.
